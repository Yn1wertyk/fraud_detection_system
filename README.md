## 🧠 Назва проекту

Дипломна робота з розробки системи виявлення фінансових махінацій з використанням машинного навчання.

## 📋 Опис проекту

Система призначена для виявлення підозрілих фінансових транзакцій. Використовує алгоритми машинного навчання (LightGBM) та методи пояснюваності (SHAP) для прийняття рішень.

## 📁 Структура проекту

## 🔧 Основні компоненти

### Інженерія ознак (features.py)
- Часові ознаки (година, день тижня, вихідні)
- Агрегації по користувачу та мерчанту
- Правила-флаги для підозрілих паттернів
- Time-aware обробка (без витоку даних)

### Модель (train.py)
- LightGBM з обробкою дизбалансу класів
- Time Series Cross Validation
- Оптимізація по PR-AUC метриці
- Візуалізація важливості ознак

### Інференс (inference.py)
- Прогнозування для окремих транзакцій
- SHAP пояснення рішень
- Бізнес-правила для прийняття рішень

### API (api.py)
- FastAPI сервіс для інтеграції
- Endpoints для окремих та пакетних запитів
- Валідація даних через Pydantic

### Веб-інтерфейс (streamlit_app.py)
- Інтерактивна перевірка транзакцій
- Пакетна обробка CSV файлів
- Візуалізація результатів та аналітика

## 📊 Метрики та оцінка
- **PR-AUC** - основна метрика (краща для дизбалансу)
- **Precision/Recall** для класу махінацій
- **F1-score** та **MCC**
- Time Series Cross Validation для валідації

## 🔍 Пояснюваність

- **SHAP** для глобальної та локальної пояснюваності
- Візуалізація внеску ознак
- Текстові пояснення рішень

## 🛡️ Безпека та етика

- Мінімізація персональних даних
- Логування всіх рішень
- Моніторинг дрейфу даних
- Оцінка справедливості (bias detection)

## 📈 Використання в продакшені

### Моніторинг
- Відстеження PR-AUC в реальному часі
- Алерти при деградації моделі
- Моніторинг дрейфу розподілів ознак

### Масштабування
- Кешування ознак для швидкості
- Асинхронна обробка пакетів
- Балансування навантаження
